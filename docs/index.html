Introduction

Option++ is a C++ facility for reading command-line program options. Option++ provides an easy way to read, validate, and process command-line options. It can also print detailed program help information.

Setup

Option++ can be incorporated into your own projects by simply placing optionpp.cpp and optionpp.hpp into your project's source directory. Then #include "optionpp.hpp" in your source file. See the example below to get started.

For detailed information about using Option++'s OptionParser class and related types, see the Class Reference section.

Example Usage

The following is a simple demonstration of how to use Option++.

      #include <iostream>
      #include <string>
      #include "optionpp.hpp"

      int main(int argc, char* argv[])
      {
        bool show_line_nums = false;
        int line_width = 16;

        OptionParser opt = {
          {'n', "line-numbers", "", "display line numbers"},
          {'w', "line-width", "WIDTH", "set maximum display width "
          "for each line"},
          {'?', "help", "", "give detailed usage information"},
        };

        opt.parse(argc, argv);

        for (const auto& o : opt) {
          switch (o.short_name) {
          case 'n':
            show_line_nums = true;
            break;
          case 'w':
            line_width = std::stoi(o.argument);
            break;
          case '?':
            opt.print_usage(std::cout);
            return 0;
          }
        }

        //...

        return 0;
      }

Argument Syntax

Option++ parses command-line arguments according to the following rules:
  • Arguments are options if they begin with a hyphen, '-'. There are two exceptions:
    • A double hyphen "--" by itself causes all remaining arguments to be interpreted as non-option arguments.
    • A single hyphen '-' by itself is interpreted as a non-option argument.
  • A single hyphen '-' specifies options by their short names, each of which consists of a single character. Multiple options can be given in one argument when short names are used.
  • A double hyphen "--" specifies a single option by its long name. A long name may consist of alphanumeric characters, hyphens, and underscores.
  • Certain options may accept an argument. Depending on the option, this argument may be mandatory or optional.
  • If an option is specified by its short name, then its argument may be given immediately afterward, or as a separate program argument, or following an equals sign '='.
  • If an option is specified by its long name, then there must be a space or equals sign '=' preceding its argument.
  • Options may appear in any order and may appear multiple times.
