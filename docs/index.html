Introduction

Option++ is a C++ library for reading command-line program options. Option++ provides an easy way to read, validate, and process command-line options. It can also print detailed program help information.

Option++ supports both long and short option names (e.g., myprogram --version or myprogram -abcd file.txt). Each program option may take a mandatory or optional argument (e.g., myprogram --color=red or myprogram -C red or even myprogram -Cred). A double dash, --, may be used by itself to indicate the end of the options, so that remaining command-line arguments do not get parsed by Option++.

Requirements

Option++ depends only on the C++ Standard Library. To compile and use Option++, your compiler must support C++11 features. Unfortunately, Option++ is not designed for projects that require compatibility with older compilers.

Setup

The Option++ library itself consists of only two files: a header file and an implementation source file. Option++ is designed to be used simply by copying these files (optionpp.cpp and optionpp.hpp) directly into your project's source directory. Then simply put #include "optionpp/optionpp.hpp" in the appropriate source file(s).

See the example below to get started.

For detailed information about using Option++'s OptionParser class and related types, see the Class Reference section.

Example Usage

The following is a simple demonstration of how to use Option++.

      #include <exception>
      #include <iostream>
      #include <string>
      #include "optionpp/optionpp.hpp"

      int main(int argc, char* argv[])
      {
        bool show_line_nums = false;
        int line_width = 16;

        optionpp::OptionParser opt = {
          {'n', "line-numbers", "", "display line numbers"},
          {'w', "line-width", "WIDTH", "set maximum display width "
          "for each line"},
          {'?', "help", "", "give detailed usage information"},
        };

        try {
          opt.parse(argc, argv);

          for (const auto& o : opt) {
            switch (o.short_name) {
            case 'n':
              show_line_nums = true;
              break;
            case 'w':
              line_width = o.arg_to_unsigned();
              break;
            case '?':
              opt.print_usage(std::cout);
              return 0;
            }
          }
        } catch (const std::exception& e) {
          std::cerr << "Error: " << e.what() << std::endl;
        }

        //...

        return 0;
      }

Argument Syntax

Option++ parses command-line arguments according to the following rules:
  • Arguments are options if they begin with a hyphen, '-'. There are two exceptions:
    • A double hyphen "--" by itself causes all remaining arguments to be interpreted as non-option arguments.
    • A single hyphen '-' by itself is interpreted as a non-option argument.
  • A single hyphen '-' specifies options by their short names, each of which consists of a single character. Multiple options can be given in one argument when short names are used.
  • A double hyphen "--" specifies a single option by its long name. A long name may consist of alphanumeric characters, hyphens, and underscores.
  • Certain options may accept an argument. Depending on the option, this argument may be mandatory or optional.
  • If an option is specified by its short name, then its argument may be given immediately afterward, or as a separate program argument, or following an equals sign '='.
  • If an option is specified by its long name, then there must be a space or equals sign '=' preceding its argument.
  • Options may appear in any order and may appear multiple times.
