struct OptionDesc

Describes a single program option, including a short and long name, a description, and an optional argument name. Is an aggregate class and can be initialized with a braced-initializer-list.

The OptionParser class is initialized with a sequence of OptionDesc objects that specify all of the command-line options that the program will accept.

Members

char short_name;
Short name (single character) for the option being described. This character can be used within the program arguments to set the option. This is done by prefixing a group of single-character option names with a '-' symbol. If this character is the null character, then the option will have no short name and must be specified by its long name.
std::string long_name;
Long name for the option, used with '--' to specify the option within the program arguments. The long name should consist of only alphanumeric characters and hyphens. In particular the long name should not include any spaces. If this string is empty, then the option will have no long name and must be specified by its short name.
std::string argument_name;
A brief name for the argument, if any, that the option takes. The argument is specified on the command line with a '=' following the option name. If this string is nonempty, Option++ will expect an argument for this option.
std::string description;
A description of the option's usage and effect on the program. For options that take an argument, this description should include information about the argument's format and usage. The descriptions for each option will make up the help message that is given by the OptionParser::usage member function.
int group;
A number that identifies the group of options to which this option belongs. Related options should be grouped together. The group numbering affects the order in which options are described in the generated help message. Group 0 (the default) is first, followed by group 1, then 2, and so on. After the nonnegative-numbered groups comes group -1, then -2, and so on.
bool arg_optional;
If the option takes an argument (that is, argument_name is nonempty), then this variable controls whether the argument is optional or mandatory. If this is false (the default), then a BadOptionArgument exception will be thrown if the option is set without an argument.
